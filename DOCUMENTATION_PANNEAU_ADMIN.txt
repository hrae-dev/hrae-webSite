───────────────────────────────────────────────────────────────────────────────
2.2 SÉCURITÉ ET MOTS DE PASSE
───────────────────────────────────────────────────────────────────────────────

RÈGLES DE SÉCURITÉ :

A) MOTS DE PASSE FORTS
   • Minimum 8 caractères
   • Mélange de majuscules et minuscules
   • Au moins un chiffre
   • Au moins un caractère spécial (@, #, $, etc.)
   • Ne pas utiliser : nom, prénom, date de naissance

   Exemples de mots de passe FORTS :
   • Hrae2024@Secure!
   • M3d!c4l$Admin
   • S4nt3#HRAE2024

B) PROTECTION DU COMPTE

   • Ne JAMAIS partager vos identifiants
   • Ne pas noter le mot de passe sur papier
   • Changez votre mot de passe tous les 3 mois
   • Déconnectez-vous après chaque session

C) PROTECTION ANTI-INTRUSION

   Le système dispose de protections automatiques :

   • LIMITE DE TENTATIVES :
     - Maximum 5 tentatives de connexion échouées
     - Après 5 échecs : compte bloqué pendant 1 heure
     - Notification à l'administrateur système

   • DÉTECTION D'ACTIVITÉ SUSPECTE :
     - Tentatives de connexion depuis des pays étrangers
     - Multiples connexions simultanées
     - Patterns d'attaque détectés

D) CHANGER VOTRE MOT DE PASSE

   1. Connectez-vous au panneau admin
   2. Cliquez sur votre nom en haut à droite
   3. Sélectionnez "Changer le mot de passe"
   4. Remplissez le formulaire :
      - Mot de passe actuel
      - Nouveau mot de passe
      - Confirmation du nouveau mot de passe
   5. Cliquez sur "Changer mon mot de passe"
   6. Confirmation : "Votre mot de passe a été changé avec succès"


───────────────────────────────────────────────────────────────────────────────
2.3 DÉCONNEXION
───────────────────────────────────────────────────────────────────────────────

COMMENT SE DÉCONNECTER :

1. Cliquez sur votre nom en haut à droite
2. Sélectionnez "Se déconnecter"
3. Confirmation : "Vous avez été déconnecté avec succès"

IMPORTANT :
• Déconnectez-vous TOUJOURS après utilisation
• Surtout si vous utilisez un ordinateur partagé
• La session expire automatiquement après 2 heures d'inactivité

L'interface utilise le thème Jazzmin avec personnalisation HRAE :

• Couleur principale : ORANGE (#FF6B35)

• Barre latérale : Fond blanc avec icônes oranges

• Services : fas fa-hospital
• Personnel : fas fa-user-md
• Articles : fas fa-newspaper
• Campagnes : fas fa-bullhorn
• Rendez-vous : fas fa-calendar-check
• Messages : fas fa-envelope

MODE CLAIR/SOMBRE :
Le système utilise le mode clair par défaut. Le mode sombre n'est pas activé
pour maintenir une cohérence visuelle avec l'identité de l'hôpital.

5.4 SUPPRIMER UN SERVICE
───────────────────────────────────────────────────────────────────────────────

ATTENTION : La suppression est DÉFINITIVE et ne peut pas être annulée !

PROCÉDURE :

Option 1 : Suppression unique
1. Ouvrez le service à supprimer
2. Faites défiler vers le bas

ANNULER UN RENDEZ-VOUS :

Raisons courantes :
• Patient ne se présente pas (no-show)
• Patient annule
• Médecin indisponible (urgence, absence)

PROCÉDURE :
1. Ouvrez le rendez-vous
2. Changez le statut en "Annulé"
3. Ajoutez une note expliquant la raison :

   "Médecin absent - patient contacté"

REPORTER UN RENDEZ-VOUS :

PROCÉDURE :

4. Ajoutez une note :
   "Reporté au 20/12/2024 à 14h - Patient d'accord"
5. Sauvegardez

IMPORTANT : Contactez toujours le patient avant de changer le statut !

10.2 RÉPONDRE AUX MESSAGES
───────────────────────────────────────────────────────────────────────────────

PROCÉDURE :

1. Ouvrez le message

2. CHANGEZ LE STATUT

   Passez de "Nouveau" à "Lu" dès que vous lisez le message

3. PRÉPAREZ VOTRE RÉPONSE

   • Lisez attentivement la demande
   • Rassemblez les informations nécessaires
   • Consultez les collègues si besoin

4. RÉPONDEZ PAR EMAIL

   Le système ne dispose pas d'envoi d'email intégré.
   Répondez manuellement :

   A) Ouvrez votre client email (Gmail, Outlook, etc.)
   B) Nouvelle message à : [email du patient]
   C) Objet : RE: [sujet du message]
   D) Rédigez votre réponse

   CONSEILS POUR LA RÉPONSE :

   • Soyez professionnel et courtois
   • Répondez précisément aux questions posées
   • Fournissez des informations complètes
   • Incluez vos coordonnées de contact
   • Signature avec nom et fonction

   EXEMPLE DE RÉPONSE :

   ```
   Objet : RE: Demande d'information sur la pédiatrie

   Bonjour Madame NKONO,

   Nous vous remercions pour votre message.

   Concernant votre demande d'information sur notre service
   de pédiatrie, voici les renseignements demandés :

   - Horaires de consultation : Lundi à Vendredi, 8h-17h
   - Tarif consultation : 5 000 FCFA
   - Médecins disponibles : Pr. NGONO, Dr. FOUDA

   Pour prendre rendez-vous, vous pouvez :
   - Appeler le +237 XXX XXX XXX
   - Utiliser notre formulaire en ligne : [lien]

   N'hésitez pas à nous recontacter pour toute question.

   Cordialement,

   [Votre nom]
   [Votre fonction]
   Hôpital Régional Annexe d'Edea
   Tel : +237 XXX XXX XXX
   Email : contact@hrae.cm
   ```

5. METTEZ À JOUR LE STATUT

   Retournez dans l'admin :
   • Changez le statut en "Répondu"
   • Ajoutez une note (optionnel) :
     "Réponse envoyée par email le 02/12/2024"
   • Sauvegardez

GESTION DES RÉCLAMATIONS :

Pour les plaintes :
• PRIORITÉ HAUTE - Répondez rapidement (< 24h)
• Prenez la plainte au sérieux
• Reconnaissez le problème
• Proposez une solution ou compensation
• Suivez le dossier jusqu'à résolution
• Informez la direction si nécessaire

GESTION DES PARTENARIATS :

Pour les propositions de collaboration :
• Transférez au responsable compétent
• Demandez plus de détails si nécessaire
• Planifiez une rencontre si intéressant
• Archivez les propositions non pertinentes

───────────────────────────────────────────────────────────────────────────────

• Exportez pour archivage externe si nécessaire

Le système enregistre automatiquement toutes les actions dans l'admin.

ACCÈS : Menu > ADMINISTRATION > Journaux (ou Recent Actions)

INFORMATIONS ENREGISTRÉES :

• Qui ? : utilisateur ayant effectué l'action
• Quoi ? : type d'action (ajout, modification, suppression)
• Quand ? : date et heure exactes
• Où ? : objet concerné (article, service, rendez-vous, etc.)

CONSULTATION :

┌─────────────────────────────────────────────────────────────────────────────┐
│  Entrées de journal                                                          │
├────────────────────────┬────────────────┬────────────┬────────────────────┤
│ Utilisateur            │ Action         │ Type       │ Date               │
├────────────────────────┼────────────────┼────────────┼────────────────────┤
│ admin                  │ Modifié        │ Article    │ 01/12/24 10:30     │
│ marie.nkono            │ Ajouté         │ Service    │ 01/12/24 09:15     │

└────────────────────────┴────────────────┴────────────┴────────────────────┘

UTILITÉ :

• Traçabilité des modifications
• Audit de sécurité
• Identification des erreurs
• Résolution de conflits
• Conformité réglementaire

SÉCURITÉ :

Le système inclut plusieurs protections :

A) PROTECTION ANTI-BRUTE FORCE (django-axes)
   • Max 5 tentatives de connexion échouées
   • Blocage de 1 heure après 5 échecs
   • Suivi par adresse IP
   • Logs des tentatives suspectes

B) RATE LIMITING (limitation de taux)
   • Formulaires publics limités :
     - Rendez-vous : 3/heure par IP
     - Contact : 5/heure par IP
     - Inscriptions campagnes : 5/heure par IP
   • Logs dans /logs/ratelimit.log

C) DÉTECTION D'ATTAQUES
   • Patterns SQL injection détectés
   • User-agents malveillants bloqués
   • Tentatives de path traversal bloquées

D) HEADERS DE SÉCURITÉ
   • HTTPS forcé
   • HSTS activé
   • Content-Security-Policy
   • X-Frame-Options: DENY
   • X-Content-Type-Options: nosniff

CONSULTATION DES LOGS :

Les logs système sont dans /logs/ :
• ratelimit.log : tentatives bloquées par rate limiting
• attack_detection.log : attaques détectées

Consultez-les régulièrement pour surveiller la sécurité.

EN CAS D'INCIDENT :

1. Identifiez l'utilisateur concerné
2. Consultez les journaux
3. Désactivez le compte si compromis
4. Changez les mots de passe
5. Informez l'administrateur système
6. Analysez les logs pour comprendre
7. Corrigez la faille si applicable

18. RÉSOLUTION DE PROBLÈMES
═══════════════════════════════════════════════════════════════════════════════

PROBLÈME : Je ne peux pas me connecter

SOLUTIONS :

1. Vérifiez votre nom d'utilisateur et mot de passe
   • Attention aux majuscules/minuscules
   • Pas d'espaces en début/fin

2. Compte bloqué après 5 tentatives ?
   • Attendez 1 heure
   • Ou contactez l'administrateur système

3. Mot de passe oublié ?
   • Contactez l'administrateur système
   • Il pourra le réinitialiser

4. Problème persistant ?
   • Essayez un autre navigateur
   • Videz le cache
   • Vérifiez votre connexion internet


PROBLÈME : L'image ne s'upload pas

SOLUTIONS :

1. Vérifiez le format
   • Acceptés : JPG, PNG, GIF
   • Pas de : BMP, TIFF, PSD

2. Vérifiez la taille
   • Maximum : 5 MB par fichier
   • Compressez si nécessaire (TinyPNG, etc.)

3. Vérifiez le nom du fichier
   • Pas de caractères spéciaux (é, à, ç, etc.)
   • Pas d'espaces (utilisez tirets ou underscores)
   • Exemple OK : service-pediatrie.jpg
   • Exemple NON : Service Pédiatrie (2024).jpg

4. Problème persistant ?
   • Essayez une autre image
   • Contactez l'administrateur système


PROBLÈME : Les modifications n'apparaissent pas sur le site

SOLUTIONS :

1. Rafraîchissez la page
   • Ctrl + F5 (Windows)
   • Cmd + Shift + R (Mac)
   • Vide le cache

2. Vérifiez le statut
   • Article : statut "Publié" ?
   • Service : case "Actif" cochée ?
   • Personnel : "Membre actif" coché ?

3. Videz le cache du site
   • Contactez l'administrateur système
   • Il peut vider le cache Redis

4. Attendez quelques minutes
   • Le cache peut prendre 5-10 minutes à se rafraîchir


PROBLÈME : Je ne peux pas supprimer un élément

SOLUTIONS :

1. Vérifiez les permissions
   • Avez-vous le droit "Supprimer" ?
   • Consultez votre groupe de permissions

2. Éléments liés
   • Django protège contre suppressions dangereuses
   • Exemple : impossible de supprimer un service si des rendez-vous y sont liés
   • Supprimez d'abord les éléments liés
   • Ou utilisez "Désactiver" au lieu de supprimer

3. Contactez un superutilisateur
   • Il pourra effectuer la suppression


PROBLÈME : L'éditeur de texte ne fonctionne pas

SOLUTIONS :

1. Vérifiez JavaScript
   • L'éditeur nécessite JavaScript activé
   • Vérifiez les paramètres de votre navigateur

2. Changez de navigateur
   • Testez avec Chrome ou Firefox

3. Désactivez les extensions
   • Bloqueurs de pub peuvent causer des problèmes
   • Testez en navigation privée

4. Contactez le support technique


PROBLÈME : Le site est lent

Ce n'est généralement PAS un problème admin, mais :

1. Vérifiez votre connexion internet

2. Images trop lourdes ?
   • Compressez avant upload
   • Taille max recommandée : 1 MB

3. Contactez l'administrateur système
   • Problème potentiel de serveur ou cache
